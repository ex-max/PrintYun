# PrintYun 项目接口说明文档

## 文档说明

本文档用于说明 `printyun-master` 项目当前已实现的主要接口，覆盖：

- 用户认证与个人中心
- 打印下单与支付
- 后台管理
- JSON/AJAX 接口
- 支付回调接口
- 测试接口

本文档基于当前代码实现整理，接口分组按 Flask Blueprint 划分。

---

## 一、项目接口总览

### 1.1 Blueprint 与前缀

| 模块 | Blueprint | URL 前缀 | 说明 |
|---|---|---|---|
| 登录与用户 | `login` | `/login` | 注册、登录、找回密码、个人中心 |
| 打印业务 | `printer` | `/printer` | 打印中心、上传文件、生成订单 |
| 后台管理 | `admin` | `/admin` | 仪表盘、订单中心、打印点管理、用户管理 |
| JSON 接口 | `jsons` | `/jsons` | AJAX 查询、XORPay 支付相关接口 |
| 支付宝 | `cloud_pay` | `/cloud_pay` | 支付宝发起支付、同步回跳、异步通知 |
| 测试接口 | `test` | `/test` | 测试上传接口 |

### 1.2 权限说明

| 权限类型 | 说明 |
|---|---|
| 公开接口 | 无需登录即可访问 |
| 登录后可访问 | 需要 `login_required` |
| 管理员接口 | 需要登录且 `current_user.Role == 'admin'` |
| 文件预览特例 | `admin` 蓝图下部分预览接口允许登录用户访问，但有归属校验 |

### 1.3 订单状态说明

| 状态值 | 含义 |
|---|---|
| `-2` | 已取消 |
| `-1` | 打印失败 |
| `0` | 未支付 |
| `1` | 已支付（兼容旧状态） |
| `2` | 正在打印 / 待取件 |
| `3` | 已完成 |

## 二、登录与用户模块 `/login`

---

### 2.1 用户注册

- **路径**：`/login/register`
- **方法**：`GET` / `POST`
- **权限**：公开
- **说明**：展示注册页并提交注册表单

#### POST 参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `tel` | int | 是 | 手机号 |
| `v_code` | int | 是 | 短信验证码 |
| `password1` | string | 是 | 密码 |
| `password2` | string | 是 | 确认密码 |

#### 返回

- `GET`：渲染注册页面 `use_templates/c_register.html`
- `POST` 成功：创建用户并跳转登录或首页
- `POST` 失败：返回原页面并显示错误

---

### 2.2 找回密码

- **路径**：`/login/findpassword`
- **方法**：`GET` / `POST`
- **权限**：公开
- **说明**：通过手机号和验证码找回密码

#### POST 参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `tel` | int | 是 | 手机号 |
| `v_code` | int | 是 | 验证码 |
| `password1` | string | 是 | 新密码 |
| `password2` | string | 是 | 确认密码 |

---

### 2.3 用户登录

- **路径**：`/login/login`
- **方法**：`GET` / `POST`
- **权限**：公开
- **说明**：用户登录入口，登录成功后按角色分流

#### POST 参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `tel` | string | 是 | 手机账号 |
| `password` | string | 是 | 密码 |
| `remember` | bool | 否 | 是否记住登录 |

#### 逻辑说明

- 已登录访问登录页时，会按角色跳转
- 若用户被禁用（`Is_Active = False`），拒绝登录
- 管理员与普通用户登录后的跳转目标不同

---

### 2.4 退出登录

- **路径**：`/login/logout`
- **方法**：`GET`
- **权限**：登录后可访问
- **说明**：退出当前登录状态

---

### 2.5 修改密码

- **路径**：`/login/change_password`
- **方法**：`GET` / `POST`
- **权限**：登录后可访问
- **说明**：用户修改当前账号密码

#### POST 参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `old_password` | string | 是 | 旧密码 |
| `password1` | string | 是 | 新密码 |
| `password2` | string | 是 | 确认新密码 |

---

### 2.6 个人中心

- **路径**：`/login/me`
- **方法**：`GET`
- **权限**：登录后可访问
- **说明**：查看当前用户资料、订单列表、订单状态统计

#### Query 参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `status` | string | 否 | 订单筛选：`all` / `unpaid` / `paid` / `done` / `cancelled` |

#### 返回内容

页面中会展示：

- 用户手机号脱敏信息
- 各状态订单数量统计
- 累计消费金额
- 最近订单列表
- 打印点中文名称映射

---

### 2.7 取消订单

- **路径**：`/login/order/<trade_number>/cancel`
- **方法**：`POST`
- **权限**：登录后可访问，且必须是订单本人
- **说明**：取消未支付订单，采用软删除

#### 路径参数

| 参数 | 说明 |
|---|---|
| `trade_number` | 订单号 |

#### 业务规则

- 仅允许取消 `Print_Status == 0` 的订单
- 取消后订单状态改为 `-2`
- 写入 `OrderLog` 审计日志

---

### 2.8 再次打印

- **路径**：`/login/order/<trade_number>/reprint`
- **方法**：`POST`
- **权限**：登录后可访问，且必须是订单本人
- **说明**：复制原订单参数生成一张新的未支付订单

#### 路径参数

| 参数 | 说明 |
|---|---|
| `trade_number` | 原订单号 |

#### 业务规则

- 复制原订单的打印参数
- 生成新的 `Trade_Number`
- 新订单默认 `Print_Status = 0`
- 写入来源订单和新订单的两条 `OrderLog`

## 三、打印业务模块 `/printer`

---

### 3.1 打印中心占位接口

- **路径**：`/printer/print`
- **方法**：`GET`
- **权限**：登录后可访问
- **说明**：当前仅返回字符串 `sss`，无实际业务意义

---

### 3.2 打印中心 / 上传下单

- **路径**：`/printer/select`
- **方法**：`GET` / `POST`
- **权限**：登录后可访问
- **说明**：打印中心核心接口，负责上传文件、计算金额、生成订单、进入支付页

#### GET 功能

- 渲染打印中心页面 `use_templates/layui_orderForGoods.html`
- 普通用户展示自己的最近订单
- 管理员展示系统最近订单

#### POST 表单参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `print_file` | file | 是 | 上传的打印文件 |
| `print_place` | string | 是 | 打印点 key |
| `print_time` | date | 否 | 预约时间 |
| `print_copies` | int | 是 | 打印份数 |
| `print_demand` | string | 是 | 打印需求：`0` 需要排版，`1` 直接打印 |
| `print_type` | string | 是 | 排版方向：`3` 竖版，`4` 横版 |
| `print_size` | string | 是 | 纸张大小，当前主要为 `A4` |
| `print_way` | string | 是 | 打印格式：单面 / 双面长边 / 双面短边 |
| `print_color` | string | 是 | 彩色样式：`CMYGray` 黑白，`RGB` 彩色 |

#### 业务逻辑

##### 情况 A：`print_demand = 0`（需要排版）

- 保存原始文件到 `static/Upload_Files/BeforeSwitchFile/`
- 创建订单
- 生成订单号 `Trade_Number`
- 返回页面 `use_templates/layui_pay_demand.html`

##### 情况 B：`print_demand = 1`（直接打印）

- 计算单价
  - 黑白单面：`0.3`
  - 黑白双面：`0.5`
  - 彩色单面：`1.0`
  - 彩色双面：`1.7`
- PDF 直接读取页数
- 图片默认按 1 页处理
- Office 类文件进入异步转换队列
- 成功后创建订单并返回支付页 `use_templates/layui_pay_form.html`

#### 可能返回页面

| 模板 | 说明 |
|---|---|
| `use_templates/layui_orderForGoods.html` | 打印中心页 / 转换失败回退页 |
| `use_templates/layui_pay_demand.html` | 需要排版的订单详情页 |
| `use_templates/layui_pay_form.html` | 直接打印的支付页 |

## 五、JSON / AJAX 模块 `/jsons`

---

### 5.1 发起 XORPay 支付

- **路径**：`/jsons/js_pay`
- **方法**：`POST`
- **权限**：登录后可访问
- **说明**：调用 XORPay 接口，生成支付二维码

#### 表单参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `tradeid` | string | 是 | 数据库订单号 |
| `pay_type` | string | 是 | 支付方式 |
| `order_id` | string | 是 | 外部订单号 |
| `order_uid` | string | 是 | 用户标识 |

#### JSON 返回示例

```json
{
  "url": "wx202604220001",
  "aoid": "xorpay-order-id"
}
```

#### 异常返回

```json
{ "mas": "XORPAY_SECRET 未配置" }
```

---

### 5.2 XORPay 支付回调

- **路径**：`/jsons/native`
- **方法**：`GET` / `POST`
- **权限**：公开
- **说明**：XORPay 回调后查询支付状态，若成功则把订单状态改为 `2`

#### 业务行为

- 查询 `aoid` 对应支付状态
- 若支付成功：
  - 更新订单为 `Print_Status = 2`
  - 写入 `OrderLog`
  - 返回 `success`

---

### 5.3 删除订单及文件

- **路径**：`/jsons/delete`
- **方法**：`POST`
- **权限**：登录后可访问
- **说明**：删除订单及其关联文件

#### 表单参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `trade_id` | string | 是 | 外部订单号，接口内部会截取真实订单号 |
| `new_filename` | string | 否 | 转换后的文件名 |
| `filename` | string | 否 | 原文件名 |

#### 权限规则

- 管理员可删除任意订单
- 普通用户只能删除自己的订单

#### JSON 返回

```json
{ "status": "success" }
```

常见错误：

```json
{ "status": "method-not-allowed" }
{ "status": "not-found" }
{ "status": "forbidden" }
```

---

### 5.4 查询支付状态

- **路径**：`/jsons/query_status`
- **方法**：`POST`
- **权限**：公开
- **说明**：查询 XORPay 订单状态

#### 表单参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `aoid` | string | 是 | XORPay 订单号 |

#### JSON 返回

```json
{ "quey_status": "success" }
```

或：

```json
{ "quey_status": "no" }
```

---

### 5.5 分页查询订单

- **路径**：`/jsons/date_times`
- **方法**：`GET` / `POST`
- **权限**：登录后可访问
- **说明**：返回订单列表 JSON 数据，支持管理员按手机号、日期筛选

#### 表单参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `tel_num` | string | 否 | 手机号 |
| `date_time` | string | 否 | 日期，格式 `YYYY-MM-DD` |
| `page` | int | 是 | 页码 |
| `limit` | int | 是 | 每页数量 |

#### 返回结构

```json
{
  "code": 0,
  "msg": "",
  "count": 12,
  "data": []
}
```

## 六、支付宝模块 `/cloud_pay`

---

### 6.1 手机端支付宝支付

- **路径**：`/cloud_pay/alipay1`
- **方法**：`GET` / `POST`
- **权限**：公开或登录后页面发起
- **说明**：生成手机端支付宝 WAP 支付链接并重定向

#### 表单参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `tradeid` | string | 是 | 数据库订单号 |

#### 返回

- 支付服务未配置：`503`
- 订单不存在：`404`

## 十、调用与维护建议

### 10.1 权限建议

- 所有后台接口应通过管理员账号访问
