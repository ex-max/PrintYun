● 自助云打印系统 (printyun) 项目详解                                                                                                                                                                                                                                                                                   
                          
  一、项目概述                                                                                                                                                                                                                                                                                                         
                                                                                                                                                                                                                                                                                                                         项目名称：自助云打印 (printyun)                                                                                                                                                                                                                                                                                        项目类型：Flask Web 应用（开源版本）                                                                                                                                                                                                                                                                                 
  业务场景：解决打印店（尤其是学校打印店）排队拥堵、U 盘传输繁琐的痛点。顾客在线上传文件 → 在线选参数 → 在线支付 → 到店凭订单号取件，打印机可根据订单信息自动打印。                                                                                                                                                    
                                                                                                                                                                                                                                                                                                      
  ▎ 注：开源仓库只包含上传/订单/支付前端系统；自动打印的监控端（监控数据库后驱动打印机）作者未开源。

  四、核心数据模型 (app/models.py)

  User
  - Id、Tel_Number（手机号作账号）、Password_Hash、Register_Date、Role（guest / admin）
  - 与 Order 一对多

  Order
  - 文件相关：File_Dir（存储名）、File_Name（原文件名）
  - 时间：Born_Date、Born_Date_Day、Print_Date（预约日期）
  - 打印参数：Print_pages、Print_Copies、Print_Direction（横/纵）、Print_Colour（CMYGray/RGB）、Print_size（A4）、Print_way（单/双面长边/双面短边）
  - 计费与状态：Print_Money、Print_Status（0 未支付 / 1 已支付 / 2 下载队列 / 3 已打印）、Trade_Number（订单号，格式 YYYYMMDD-递增序号）
  - Time_Way：0 = 上传后到店排版，1 = 在线直接打印

  六、文件转换流程（亮点）

  1. 用户上传 .doc/.docx/.ppt/.xls 等 → 先存 static/Upload_Files/BeforeSwitchFile/
  2. que.enqueue(switch_topdf, path, user_id) 把任务丢进 RQ 队列
  3. worker.py 的 RQ Worker 调用本机 soffice --headless --convert-to pdf 生成 PDF 到 static/Upload_Files/
  4. 完成后通过 Redis Pub/Sub 以 current_user.Id 为频道发布 OK/None
  5. printer.select 视图里 ps.listen() 阻塞等结果 → 读取 PDF 页数 → 计算总价

  ▎ PDF、JPG、PNG 文件不走转换流程，直接存盘读页数。

  ---
  七、支付流程

  1. js_pay：按 Trade_Number 查订单 → 组装 name/pay_type/price/order_id/notify_url → MD5 签名 → POST 到 xorpay.com/api/pay/xxxx → 返回二维码链接 → qrcode 生成二维码保存到 static/pay_qrcode/
  2. 前端轮询 query_status → 拿 aoid 去 xorpay.com/api/query/ 查询
  3. 用户扫码付款后，xorpay 回调 /jsons/native → 再次查询确认 → 把订单 Print_Status 置 1
  4. 支付宝相关逻辑在 app/test/ali_pay.py（需配 certs/ 下 3 个密钥）

  ---
  八、部署方式

  - 本机：改 config.py（MySQL）、sms.py（短信）、certs/（支付宝）→ pip install -r requirements.txt → flask run → 另起 python worker.py（异步转 PDF + 支付）
  - Docker：docker build -t printyun . → docker run -d -p 8001:8001 printyun；内部由 supervisord 通过 printyun.conf 同时拉起 Flask（Gunicorn）和 worker 进程

  九、需注意的问题（按代码直观观察）

  1. 版本陈旧：Flask 1.1.1、Werkzeug 0.15、PyPDF2 1.26 都已过时；PdfFileReader/.numPages 在新版 PyPDF2 会报错
  2. 安全隐患：
    - config.py 默认 SECRET_KEY='SECRET KEY'，数据库密码明文 root:123456
    - json.py 里签名 key 55ab2edb80e... 硬编码
    - admin.look_picture / look_media 直接把路径参数传给模板，存在目录穿越/越权访问风险
    - /jsons/delete 无登录保护、无 CSRF 验证、直接按表单传入的订单号删库删文件
  3. 阻塞 I/O：printer.select 用 Redis Pub/Sub 在 HTTP 请求内同步 listen() 等转换结果，高并发会耗尽 Gunicorn worker
  4. 表单字段名：app/control/login.py:22 里 phone_number = request.args.get('mobile_phone_number') 与前端字段需对齐
  5. README 说明：线上测试地址 http://139.9.7.123:8001/login/login（账号 123456 / 密码 123456）未装 Redis，无法测支付

  ---
  十、一句话总结

  这是一个 2019 年的 Flask + Layui 教学级项目，演示了「上传 → Office 转 PDF → 下订单 → 在线支付 → 后台管理」完整链路；思路清晰但代码质量、依赖版本和安全性都需要现代化改造才能投入生产。如果你要基于它二次开发，建议先升级依赖 + 修复安全问题 + 把支付/转换改成异步解耦。
